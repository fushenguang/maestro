## 本教程写给谁

**写给会写代码、用过终端、日常写项目——但没让 AI 帮你写过代码的人。**

如果你符合以下描述，这是给你的：
- 你知道 `npm install` 是什么，用过 git，在终端里不会迷路
- 你听说 Claude Code 很强，但不知道它到底能帮你做什么
- 你打开过一次 CC，说了句"帮我写个网站"，然后被它生成的一堆文件吓到了

如果你不符合——也没关系：
- **完全没写过代码？** 本教程不适合你。先学 JavaScript 基础，再回来。
- **已经在用 CC 做日常开发了？** 直接跳到你想深入的那章——不用从头读。
- **你是团队负责人，要评估 CC 能不能上生产？** 跳到「团队协作」和「企业部署」篇。

## 教程怎么组织

14 个主题篇 + 准备篇 + 附录。但不是让你从头读到尾。

每篇有一个情绪锚点——不是你"应该学什么"，是你"会遇到什么"：

| 篇 | 你在这篇的状态 | 你会做出什么 |
|---|---|---|
| 准备篇 | 你还没开始，想先把环境搭好 | 装好工具、配通网络、收藏免费资源 |
| 快速上手（01）| 你刚装上 CC，想知道它能干嘛 | 一个能发给朋友的贪吃蛇游戏 |
| 配置 CC（02）| 你在用出厂设置裸跑，想调优 | 一套生产级 settings + 模型代理 + superpowers |
| Skill 系统（03）| 你反复教 CC 做同一件事，想让它记住 | 可复用的 Skill 资产库 |
| MCP 协议（04）| CC 说"我访问不了数据库" | 让 CC 连接外部 API 和数据源 |
| Memory 系统（05）| 每次新开 session 都要重讲项目规范 | 跨 session 的项目记忆系统 |
| Context 管理（06）| 长会话中 CC 开始"忘记"之前说了什么 | Token 优化 + 分段策略 |
| Hooks 系统（07）| CC 改代码没验证，上线就炸 | 自动检查 + 安全拦截 + 工作流自动化 |
| Sub-agents（08）| 一个 CC 不够用，想让它并行干活 | 多 Agent 编排 + Worktree 隔离 |
| 团队协作（09）| 从单人用到 3-5 人团队用 | 共享规范 + AI Code Review + CI/CD |
| 最佳实践（10）| CC 写代码很快但越来越乱 | Prompt 工程 + Git 工作流 + 社区模式 |
| 插件系统（11）| 想扩展 CC 的能力边界 | 安装/定制/发布 CC 插件 |
| 企业部署（12）| 老板问"CC 安全吗？合规吗？" | 安全策略 + 合规审计 + SSO |
| 项目实战（13）| 想看 CC 做完整项目的全过程 | 8 个完整项目，从桌面应用到游戏引擎 |
| 生态 & 工具（14）| 想知道 CC 在行业中的位子 | 横向对比 + SDK 集成 + 社区资源 |

## 4 条阅读路径

按你现在最想做的事，选一条：

### 路 A：先做出个东西再说 🎮

你刚装好 CC（或者还没装），不想看概念——想做一个能玩的东西。

**准备篇 → 快速上手。** 先确保环境和工具就绪，然后 4 章内做出贪吃蛇。
做完之后跳到「项目实战」篇的 Tauri 桌面版，把网页版升级成双击就开的桌面应用。

### 路 B：我已经在用 CC，遇到了具体问题 🎯

按症状跳章。不需要按顺序。

| 你遇到的情况 | 去哪篇 |
|---|---|
| CC 乱改代码、改了你没让改的 | Hooks 系统（07）|
| 每次新开 session 都要重新解释项目规范 | Memory 系统（05）|
| 反复教 CC 做同一件事，想让它记住 | Skill 系统（03）|
| CC 说"我没有数据库权限" | MCP 协议（04）|
| 长会话中 CC 开始"忘事" | Context 管理（06）|
| 一个 CC 不够用，想让它并行干活 | Sub-agents（08）|
| 想看 CC 做完整项目的全流程 | 项目实战（13）|
| 代码越来越乱，想建立工程纪律 | 最佳实践（10）|
| 团队想统一用 CC，不知道怎么管 | 团队协作（09）|
| 想知道 CC 和其他工具哪个适合我 | 生态 & 工具（14）|

### 路 C：我要系统掌握 CC 📚

你有时间、有耐心，想从头建立完整的心智模型。

**准备篇 → 快速上手 → 配置 → Skill → MCP → Memory → Context → Hooks → Sub-agents → 团队协作 → 最佳实践 → 插件 → 企业部署 → 项目实战 → 生态。**

但注意：每个主题都分**基础**（学会高效使用）和**深入**（理解原理）两部分。先过基础，深入部分按你的需求选择性读——不需要每篇都啃完。

### 路 D：我是团队负责人，要评估 CC 🏢

你关心安全、合规、成本。不关心怎么写 Skill——关心这东西能不能上生产。

**直接跳到：**
- 配置 CC（02）——了解权限和模型策略
- 团队协作（09）——3-5 人团队怎么用 CC
- 企业部署（12）——安全模型、合规、SSO
- 生态 & 工具（14）——CC vs 竞品对比 + SDK 集成

## 每章的结构

每章聚焦一件事。结构是固定的：

1. **一个真实场景**——让你觉得"这确实是我会遇到的问题"
2. **让你先撞墙**——试一下，发现不对劲
3. **给答案**——CC 的哪个功能解决这个问题
4. **讲原理**——为什么这个功能这样设计
5. **留白**——一个你必须自己动手完成的东西

如果你发现一章不符合这个结构——那是 bug。

## 关于模型和平台

本教程统一使用 **DeepSeek V4 Pro** 作为演示模型——性价比最高的选择，中国大陆直接访问。

教程代码覆盖 **macOS / Windows / Linux** 三平台。终端命令会标注平台差异。

Claude Code 版本：**v2.1.176**。不同版本的行为可能有差异——如果你用的是更新或更旧的版本，以实际行为为准。
