## 本教程写给谁

**写给会写代码、用过终端、日常写项目——但没让 AI 帮你写过代码的人。**

如果你符合以下描述，这是给你的：
- 你知道 `npm install` 是什么，用过 git，在终端里不会迷路
- 你听说 Claude Code 很强，但不知道它到底能帮你做什么
- 你打开过一次 CC，说了句"帮我写个网站"，然后被它生成的一堆文件吓到了

如果你不符合——也没关系：
- **完全没写过代码？** 本教程不适合你。先学 JavaScript 基础，再回来。
- **已经在用 CC 做日常开发了？** 直接跳到你想深入的那章——不用从头读。
- **你是团队负责人，要评估 CC 能不能上生产？** 跳到第 20 章。

## 教程怎么组织

23 章，分 6 篇。但不是让你从头读到尾。

每篇有一个情绪锚点——不是你"应该学什么"，是你"会遇到什么"：

| 篇 | 你在这篇的状态 | 你会做出什么 |
|---|---|---|
| 篇一（01-04）| 你刚装上 CC，想知道它能干嘛 | 一个能发给朋友的贪吃蛇游戏 |
| 篇二（05-09）| 你的项目出问题了，想搞清楚原理 | 理解 CC 的 5 个核心机制 |
| 篇三（10-12）| 你想把 Web 项目做成桌面应用 | 3 个 Tauri 桌面应用 |
| 篇四（13-15）| 你想用 CC 赚钱、给家人做工具 | 付费 Newsletter + 学习工具 |
| 篇五（16-18）| CC 写代码很快但越来越乱 | 工程方法论——让 CC 快且可靠 |
| 篇六（19-23）| 你想知道 CC 在全行业的位子 | Rust 进阶 + 企业部署 + 生态参与 |

## 4 条阅读路径

按你现在最想做的事，选一条：

### 路 A：先做出个东西再说 🎮

你刚装好 CC（或者还没装），不想看概念——想做一个能玩的东西。

**从 01 开始 → 02 → 03 → 04。** 4 章内做出贪吃蛇。
然后跳到 10（Tauri 入门），把网页版升级成桌面应用。

### 路 B：我已经在用 CC，遇到了具体问题 🎯

按症状跳章。不需要按顺序。

| 你遇到的情况 | 去哪章 |
|---|---|
| CC 乱改代码、改了你没让改的 | 05（Hooks 深度）|
| 每次新开 session 都要重新解释项目规范 | 06（Memory 系统）|
| 反复教 CC 做同一件事，想让它记住 | 07（Skill 实战）|
| CC 说"我没有数据库权限" | 08（MCP）|
| 一个 CC 不够用，想让它并行干活 | 09（Sub-agents）|
| 代码越来越乱，想建立工程纪律 | 16-18（工程实践）|

### 路 C：我要系统掌握 CC 📚

你有时间、有耐心，想从头建立完整的心智模型。

**00 → 01 → 02 → ... → 23。** 按顺序读。

但注意："按顺序"不是"概念从易到难"——是**项目从简单到复杂**。03 章做网页贪吃蛇，10 章升级成桌面应用，13 章开始收钱，17 章用 Rust 写游戏引擎。顺序本身是一根"你越来越强"的弧线。

### 路 D：我是团队负责人，要评估 CC 🏢

你关心安全、合规、成本。不关心怎么写 Skill——关心这东西能不能上生产。

**直接跳到 20 → 21 → 23：**
- 20（企业部署 & 治理）——安全模型、合规、审计日志
- 21（同类工具横向对比）——CC vs Cursor vs Codex vs Aider
- 23（Agent SDK & API 集成）——把 CC 嵌入 CI/CD

## 每章的结构

每章聚焦一件事。结构是固定的：

1. **一个真实场景**——让你觉得"这确实是我会遇到的问题"
2. **让你先撞墙**——试一下，发现不对劲
3. **给答案**——CC 的哪个功能解决这个问题
4. **讲原理**——为什么这个功能这样设计
5. **留白**——一个你必须自己动手完成的东西

如果你发现一章不符合这个结构——那是 bug。

## 关于模型和平台

本教程统一使用 **DeepSeek V4 Pro** 作为演示模型——性价比最高的选择，中国大陆直接访问。

教程代码覆盖 **macOS / Windows / Linux** 三平台。终端命令会标注平台差异。

Claude Code 版本：**v2.1.174**。不同版本的行为可能有差异——如果你用的是更新或更旧的版本，以实际行为为准。
