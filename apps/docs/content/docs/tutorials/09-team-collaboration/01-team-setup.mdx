**配项目**：为一个 3 人团队设计 CC 接入方案——每个人什么时候用 CC、什么时候自己写、代码怎么合入。

## 计划覆盖

- 团队 CC 工作流的 3 种模式：每人一个 CC / 共享 CC 实例 / CI 中的 CC
- 角色分工：谁写 prompt、谁 review CC 生成的代码、谁维护 Skill 库
- 3-5 人团队的 CC 使用节奏：不是所有人都同时用
- 反模式：全员同时让 CC 改同一个 repo——Worktree 冲突风暴
