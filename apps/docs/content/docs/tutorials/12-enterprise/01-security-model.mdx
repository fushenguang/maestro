**配项目**：为你的团队设计一套 CC 安全策略——最小权限矩阵 + Hook 白名单 + 审计日志模板。

## 计划覆盖

- 0 信任模型：不信任任何人——包括写好 hook 的你自己
- 个人 vs 团队：个人用 CC = 自己的风险自己担；团队用 CC = 一个人的错误全团队买单
- 最小权限矩阵：按角色（开发者 / TL / DevOps）定义权限
- 审计追踪：谁、什么时候、让 CC 做了什么
