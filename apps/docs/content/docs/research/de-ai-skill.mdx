# De-AI 写作 Skill

## Card（速览）

### 一句话定位

> 一个 Claude Code skill，检测并消除中文技术写作中的 AI 生成痕迹，同时保留写作者的句法指纹。四步工作流：判断动不动 → 检测 AI 味（五组 20+ 模式）→ 自审计 → 交代改动。

### 关键 takeaway

1. **三层架构**：哲学层（来自 renwei——少动、保手迹、隐形是上限）+ 检测层（来自 humanizer——五组 20+ 中文化 AI 模式 + 聚类判断 + 误伤防护）+ 审计层（双轮重写：Draft → "哪里还像 AI 写的？" → Final）。三层各有来源、各有分工。(SKILL.md:L1-L89)

2. **教程写作专项优化**：识别出教程场景 Top 6 高频 AI 味——签到过渡、碎片标题、谄媚鼓励、假坦诚开场、排比三连、万能结尾。准确性 > 文采。代码注释豁免去味。技术术语保留英文原文。技术教程的"人味"不靠故事——靠作者的真实踩坑、技术偏好、不完美的类比。(SKILL.md:L90-L115)

3. **中文化适配策略**：humanizer 的 33 种英文 AI 模式 → 高适配 18 种（可直接移植）、需适配 10 种（中文表现不同）、低适配 5 种（英文特有）。核心工作：AI 词汇黑名单中文化 + 中文特有模式补充（四字格堆砌、文言虚词滥用、成语串烧、"的"字句尾）。(humanizer Deep-Dive 主题 1)

4. **误伤防护是第一优先**：12 种不构成 AI 证据的情况（完美语法、单个破折号、中英混排等）+ 6 种真人写作保真信号（具体细节、未解决的矛盾、句子呼吸、自我打断、技术偏好、不完美类比）。判断标准是聚集，不是孤立信号。(SKILL.md:L117-L138)

5. **已在 6 篇教程上跑过一遍审查**：动 9 处的建议（4 处高把握 + 3 处中把握 + 2 处拿不准）。0.36% 的改动率，远低于警戒线——说明这批教程原始质量高、人味足。验证了"说不出来打绊了就不改"的原则在实践中不会导致"什么都不改"——它会在真正需要改的地方精准下手。(de-ai review 2026-06-13)

### 与教程写作流程的关联

- **写新章节前**：跑 `/de-ai` 检查大纲 → 给每节的标题和开头两句做 pre-audit，防 AI 味从结构层渗入
- **写完后**：跑 `/de-ai` 检查全文 → 对照五组模式 + Top 6 教程高频 AI 味
- **团队 review**：拿 de-ai 的检测报告作为 code review 的一个 check —— "这章有 3 处签到过渡、2 处碎片标题" 比 "感觉有点 AI 味" 具体得多
- **迭代**：每次翻车 → 追加 gotcha → skill 进化。Gotchas 区已经从两个源各继承了几条，后续每发现一个新模式或新盲区就加一条
