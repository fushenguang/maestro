# Memory 与 CLAUDE.md（官方文档）

## 一句话定位

> Claude Code 的「跨会话知识」由两套互补机制承担：CLAUDE.md（人写的持久指令，4 层加载）+ auto memory（Claude 自己累积的学习笔记）；两者都在每场会话开始时进入 context，但都是「指南」而非「强制」——硬约束要用 hook。

## 关键 takeaway

1. **CLAUDE.md ≠ auto memory，两套都加载**：CLAUDE.md 由人写、范围 project / user / org、整段注入；auto memory 由 Claude 写、范围 per-repo、只截 MEMORY.md 前 200 行或 25KB 注入。两者作为 user message 进入 context（**不是** system prompt），不强制——硬阻断动作用 `PreToolUse` hook。锚点 `docs:L1:14, L1:1736`。
2. **CLAUDE.md 4 级加载链**（从广到窄）：**Managed policy**（org 级，IT 部署，不可个人 exclude）> **User**（`~/.claude/CLAUDE.md`）> **Project**（`./CLAUDE.md` 或 `./.claude/CLAUDE.md`，进版本控制）> **Local**（`./CLAUDE.local.md`，**必须 gitignore**）；多文件按目录树「从根到 cwd」**concatenate** 而非 override。锚点 `docs:L1:80, L1:10237`。
3. **子目录懒加载 + HTML 注释剥离**：cwd **之上** 的 CLAUDE.md 启动即注入；cwd **之下** 的子目录 CLAUDE.md 仅在 Claude 读取该子目录文件时才挂入——monorepo 噪声控制的关键。`<!-- maintainer notes -->` 这种块级 HTML 注释在注入前被剥掉（code 块内保留）。锚点 `docs:L1:11179, L1:11459`。
4. **`.claude/rules/` 拆分大文件 + `paths:` 域**：`testing.md` / `api-design.md` 等小文件加 `paths: src/api/**/*.ts` glob 后，**仅在 Claude 处理匹配文件时**挂入；不加 paths 则全量加载。配合 `@path/to/import` 跨文件引用（递归深度上限 4 跳）+ `~/.claude/rules/` 用户级规则 + AGENTS.md 桥接（`@AGENTS.md` 或 symlink）。锚点 `docs:L1:172, L1:196, L1:312, L1:13872`。
5. **auto memory 存 `~/.claude/projects/<project>/memory/`**：所有 worktree 与子目录共享同一 memory dir；MEMORY.md 当索引，前 200 行 / 25KB 注入。详细 topic 文件不预载，按需 Read。可用 `autoMemoryEnabled: false` 或 `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1` 关闭，`autoMemoryDirectory` 可改路径。**机器本地，不跨机同步**。锚点 `docs:L1:551, L1:19465, L1:19364`。
6. **`/memory` 命令 + `/compact` 行为差异**：`/memory` 列当前 session 加载的所有 CLAUDE.md / rules + 切 auto memory 开关。`/compact` 后**根目录 CLAUDE.md 重新读盘并 re-inject**，**子目录 CLAUDE.md 不会自动 re-inject**——「compact 后某指令消失」基本是这两种情况之一。锚点 `docs:L1:591, L1:24563`。

## Brief（梳理）

### Memory 双层机制

- **CLAUDE.md = 人写、整段注入**（指南；硬约束要用 hook）
- **auto memory = Claude 写、按行截断注入**（学习笔记）
- **两者都进 context**（user message，**不是** system prompt）
- 都不强制——硬阻断动作用 `PreToolUse` hook

### CLAUDE.md 4 级加载链（从广到窄）

```
Managed policy  >  User  >  Project  >  Local
（IT 部署）         (~/.claude/)  (./CLAUDE.md)  (./CLAUDE.local.md)
                                   提交 git      gitignore
```

- **多文件**：从根到 cwd **concatenate**（不是 override）
- **Managed policy 不可个人 exclude**——IT 部署的最高级

### 子目录懒加载 + HTML 注释剥离

- **cwd 之上**：启动即注入
- **cwd 之下**（子目录 CLAUDE.md）：**仅在 Claude 读该子目录文件时挂入**——monorepo 噪声控制关键
- `<!-- maintainer notes -->` 这种 HTML 注释在注入前被剥掉（code 块内保留）

### `.claude/rules/` 拆分大文件

- `testing.md` / `api-design.md` 等小文件加 `paths: src/api/**/*.ts` glob 后，**仅在 Claude 处理匹配文件时挂入**
- 不加 `paths` 则全量加载
- **跨文件引用**：`@path/to/import`（递归深度上限 4 跳）
- **AGENTS.md 桥接**：`@AGENTS.md` 或 symlink

### auto memory 存哪

- `~/.claude/projects/<project>/memory/`（机器本地，**不跨机同步**）
- 所有 worktree 与子目录共享同一 memory dir
- **MEMORY.md 是索引**（前 200 行 / 25KB 注入）
- 详细 topic 文件不预载，按需 Read
- 关闭：`autoMemoryEnabled: false` 或 `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1`
- 改路径：`autoMemoryDirectory`

### `/memory` vs `/compact` 行为差异

| 命令 | 行为 |
|---|---|
| `/memory` | 列当前 session 加载的所有 CLAUDE.md / rules + 切 auto memory 开关 |
| `/compact` 后 | **根目录 CLAUDE.md 重新读盘 re-inject**；**子目录 CLAUDE.md 不会自动 re-inject** |

> 「compact 后某指令消失」基本是这两种情况之一

### 与其他源的关系

- **sub-agents**（`memory: user / project / local` 字段）：sub-agent 自己的 memory，与本节 CLAUDE.md / auto memory 是**两套并存**
- **Agent Skills**：按需挂载（与 CLAUDE.md 加载时机不同）
- **Settings**：`autoMemoryEnabled` / `claudeMdExcludes` 控制开关
- **hooks**：本节是「指南」、hook 是「强制」——核心区分

### 实战组合（推荐默认配置）

```text
managed policy        → 锁安全基线（IT 部署）
项目 CLAUDE.md         → 工程规约（commit 进 git）
.claude/CLAUDE.local.md → 个人 sandbox（gitignore）
auto memory           → 沉淀踩坑笔记
.claude/rules/        → path-scoped 拆 API / DB / testing 规则
                       降 context 占用
```

### 跨源关联

- vs `anth-hooks`：本 = 指南；hooks = 强制——**错位互补**（CLAUDE.md 给「应该怎么做」、hooks 给「必须怎么做」）
- vs `anth-sub-agents`：sub-agent 有自己的 `memory: user/project/local` 字段——**并列存在**（不与本节共享）
- vs `superpowers`：superpowers 的 14 skill 体系用 auto memory 路径做知识沉淀——**互补**：auto memory 跨会话 + skill 按需挂载
