# Sub-agents（官方文档）

## 一句话定位

> Claude Code 的子智能体（subagent）是带独立上下文窗口 + 自定义系统提示 + 受限工具/权限/模型的专用 AI 助手，主会话按 `description` 自动委派或显式 `@`-mention 调用。

## 关键 takeaway

1. **核心收益 = 隔离上下文 + 复用配置**：子智能体在独立 context window 中工作，把搜索结果、日志、文件内容等高噪声输出拦在子任务里，只把摘要回传主对话；同时可在 `~/.claude/agents/`（个人）或 `.claude/agents/`（项目）复用定义，配合 `memory: user/project/local` 跨会话沉淀知识。锚点 `docs:L2, L5, L11`。
2. **配置面 = Markdown + YAML frontmatter**：`name`、`description` 必填，其余可选 `tools` / `disallowedTools` / `model` / `permissionMode` / `mcpServers` / `hooks` / `skills` / `memory` / `background` / `isolation: worktree` / `effort` / `color` 等；描述写明 `Use proactively` 之类的措辞可促进自动委派。
3. **关键不变量 = 不能嵌套 spawn**：「subagents cannot spawn other subagents」是硬约束（防无限嵌套），需要多步链路时由主会话链式串联；Agent tool 上的 `Agent(agent_type)` 语法只对主线程生效。锚点 `docs:L26, L138`。
4. **内置 3 + 1 类**：Explore（Haiku 只读，做代码搜索）、Plan（plan 模式下做只读研究，避免与主循环互相递归）、general-purpose（继承主会话模型与全部工具，处理多步复杂任务），另含 `statusline-setup`、`claude-code-guide` 等辅助；可经 `permissions.deny` 屏蔽单个或 `deny Agent` 全局禁用，SDK 侧可用 `CLAUDE_AGENT_SDK_DISABLE_BUILTIN_AGENTS=1`。锚点 `docs:L17`（Explore/Plan 跳过 CLAUDE.md 与 parent git status 的设计意图）。
5. **Foreground vs Background & Fork 三种委派模式**：前台子任务阻塞主对话并透传权限请求；后台子任务并发执行并 auto-deny 需提示的调用；fork（`CLAUDE_CODE_FORK_SUBAGENT=1`）则继承主会话完整历史 / 系统提示 / 模型 / 权限，但自己的工具调用仍隔离，并通过共享 prompt cache 降低成本。锚点 `docs:L165, L208`。

## Brief（梳理）

### Sub-agents 核心收益

- **隔离上下文**：子任务在独立 context window 跑，把搜索 / 日志 / 文件等高噪声输出拦在子任务里，只摘要回传主对话
- **复用配置**：`~/.claude/agents/`（个人）或 `.claude/agents/`（项目）跨项目复用
- **持久化记忆**：`memory: user / project / local` 跨会话沉淀知识（如「本项目用 uv 不用 pip」）
- **cost routing**：子任务用小模型（Haiku）做廉价检索，主对话用大模型做关键决策

### 配置面：Markdown + YAML frontmatter

**必填**：`name`、`description`（**描述写明 `Use proactively` 可促进自动委派**）

**可选字段**：
- `tools` / `disallowedTools`：限定可用工具
- `model`：sonnet / opus / haiku
- `permissionMode`：plan / bypassPermissions / default
- `mcpServers`：子任务可用的 MCP server
- `hooks`：子任务内的 hook 配置
- `skills`：子任务可调 skill
- `memory` / `background` / `isolation: worktree` / `effort` / `color`

### 关键不变量：不能嵌套 spawn

- **硬约束**：「subagents cannot spawn other subagents」——防无限嵌套
- **解决**：需要多步链路时由主会话**链式串联**多个子任务
- **Agent tool 上的 `Agent(agent_type)` 语法只对主线程生效**——子任务内不能调 Agent 工具

### 内置 3 + 1 类

| 类型 | 用途 | 关键设计 |
|---|---|---|
| **Explore** | 代码搜索（只读）| Haiku 模型；**跳过 CLAUDE.md 与父 git status**（防干扰）|
| **Plan** | plan 模式下只读研究 | 避免与主循环互相递归 |
| **general-purpose** | 多步复杂任务 | 继承主会话模型与全部工具 |
| **辅助** | statusline-setup / claude-code-guide | UI 辅助 + 文档检索 |

- **屏蔽**：`permissions.deny` 屏蔽单个；`deny Agent` 全局禁；SDK 用 `CLAUDE_AGENT_SDK_DISABLE_BUILTIN_AGENTS=1`

### 3 种委派模式

| 模式 | 行为 | 适用 |
|---|---|---|
| **Foreground** | 阻塞主对话；透传权限请求 | 短任务；强依赖子任务结果 |
| **Background** | 并发执行；auto-deny 需提示的调用 | 长任务；多任务并行 |
| **Fork** | 继承主会话完整历史 / 系统提示 / 模型 / 权限；工具调用隔离；**共享 prompt cache 降本** | 「实验性 / 探索性」分支 |

- **Fork 触发**：`CLAUDE_CODE_FORK_SUBAGENT=1` env var

### 跨源关联

- vs `anth-agent-sdk`：SDK 在 agent 进程内**可**用 `Agent(agent_type)`——但子任务内仍不能嵌套 spawn——**沿用不变量** + 错位（SDK 给「程序化编排」、sub-agents 给「声明式配置」）
