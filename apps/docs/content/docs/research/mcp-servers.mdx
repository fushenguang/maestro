# mcp-servers

## 一句话定位

> Anthropic 官方维护的 MCP **参考实现集合**——7 个 small server（4 TS + 3 Py），演示 Tools/Resources/Prompts/Roots 用法并作 SDK 样板；生产集成请去 MCP Registry。

## 关键 takeaway

1. **reference，不是产品目录**——README 显式「not production-ready」(skill_collection:L9)；完整 server 列表走 MCP Registry (skill_collection:L6)。
2. **monorepo 装 7 个 server**（4 TS + 3 Py），各自独立 npm/PyPI 包：everything / fetch / filesystem / git / memory / sequentialthinking / time (skill_collection:L29-L35)。
3. **13 个老 server 已归档**到 servers-archived（含 github、postgres、gdrive、puppeteer、redis、sentry、slack、sqlite、brave-search 等）(skill_collection:L41-L53)。
4. **零 clone 跑**：`npx -y @modelcontextprotocol/server-memory` 或 `uvx mcp-server-git` (skill_collection:L61-L75)。
5. **真用靠 MCP client config**——`mcpServers` 写 `command`+`args`+`env`；Windows 用 `cmd /c` 包 npx (skill_collection:L80-L134)。
6. **CI/SDK 范式**——TS：`npm workspaces` + `tsc` + `vitest`（Node 22）；Py：`uv` + `hatchling` + `pyright` + `ruff`（Py ≥ 3.10）(skill_collection:CLAUDE.md)。
7. **Tool annotations**——filesystem server 在 tool 上声明 `readOnlyHint` / `idempotentHint` / `destructiveHint` (skill_collection:filesystem/README.md:L189-L205)。
8. **License 双轨**：MIT（旧）+ Apache-2.0（新），文档 CC-BY-4.0 (skill_collection:L154, LICENSE:L1-L5)。

## Brief（梳理）

### 7 个 reference server 详表

| Server | 语言 | NPM/PyPI 包 | 核心用途 | 关键 tool / resource |
|---|---|---|---|---|
| **everything** | TS | `@modelcontextprotocol/server-everything` | **测试 / 演示**——展示所有 MCP 特性（prompts / resources / tools） | 完整特性样板 |
| **fetch** | Py | `mcp-server-fetch` | 网页内容抓取 + 转 markdown（为 LLM 用）| `fetch` tool |
| **filesystem** | TS | `@modelcontextprotocol/server-filesystem` | **安全文件操作**——可配置允许目录 + Tool annotations | 11 个 verb-first tool（`read_text_file` / `write_file` / `move_file` 等）|
| **git** | Py | `mcp-server-git` | 读 / 搜索 / 操作 git 仓库 | `git_log` / `git_diff` / `git_commit` 等 |
| **memory** | TS | `@modelcontextprotocol/server-memory` | **知识图谱持久化** | entity + relation CRUD |
| **sequentialthinking** | TS | `@modelcontextprotocol/server-sequential-thinking` | **动态反思**——thought sequences 推进 | `sequential_thinking` 步骤化推理 |
| **time** | Py | `mcp-server-time` | 时区 + 时间转换 | `get_current_time` / `convert_time` |

- **2 维度观察**：
  - **语言分布**：4 TS（更重 SDK 集成）+ 3 Py（轻量单职责）
  - **角色分布**：3 个 I/O 工具（fetch / filesystem / git）+ 2 个状态工具（memory / sequentialthinking）+ 1 个 meta 工具（time）+ 1 个测试工具（everything）

### 13 个 archived server 类别

`servers-archived` 13 个老 server（`README.md:L41-L53`）：AWS KB Retrieval / Brave Search / EverArt / GitHub / GitLab / Google Drive / Google Maps / PostgreSQL / Puppeteer / Redis / Sentry / Slack / SQLite——多数被**官方新版或第三方**取代（如 Slack 移到 Zencoder / Brave Search 独立官方仓库 / GitHub/GDrive 继续官方维护）。

- **重要信号**：MCP 官方**主动**让 reference 集合**小而精**——把"特定服务"交给 **Registry**（`registry.modelcontextprotocol.io`），不让 reference 仓库膨胀。

### monorepo 工程结构（CLAUDE.md 全档）

**目录结构**（`CLAUDE.md:L10-L20`）：
```

  filesystem/         TS  @modelcontextprotocol/server-filesystem
```

**双栈工程实践**：

| 维度 | TypeScript | Python |
|---|---|---|
| **模块系统** | ESM + `.js` 后缀 import | 标准 async/await |
| **类型校验** | `tsc` strict mode (ES2022 / Node16) | `pyright`（CI 强制）|
| **测试** | `vitest` + `@vitest/coverage-v8` | `pytest` + `pytest-asyncio`（fetch）|
| **Lint** | ESLint 风格 | `ruff` |
| **打包** | `tsc` 输出 | `hatchling` + `uv build` |
| **依赖管理** | `npm workspaces` | `uv`（非 pip）|
| **Node / Python 版本** | Node 22 | Python >= 3.10（per-server `.python-version`）|

**CI/CD 流水线**（`CLAUDE.md:L88-L93`）：`detect-packages`（find + jq 矩阵）→ `test` → `build` → `publish`（release 时 npm / PyPI trusted publishing）。

### 关键 SDK 范式

**Tool annotations 矩阵**（filesystem README L189-L205，**必读**——MCP 协议级样板）：

| Tool | readOnlyHint | idempotentHint | destructiveHint | 备注 |
|---|---|---|---|---|
| `read_text_file` | true | – | – | Pure read |
| `write_file` | false | true | true | Overwrites existing files |
| `move_file` | false | false | true | Deletes source file |

- **不变量**：`idempotentHint` 和 `destructiveHint` 仅在 `readOnlyHint: false` 时有意义（spec 定义）。
- **设计价值**：让 client 端 LLM 在调用前**预判**风险，配合 approval policy（CC `permission-mode`、Codex `approval`）实现细粒度拦截。

**注册函数**（`CLAUDE.md:L101-L104`）：
- `registerTools(server)` / `registerResources(server)` / `registerPrompts(server)` 三个分离入口
- **约定**：每个 server 必须**演示** Resources + Prompts + Roots（CLAUDE.md:L80 「not just Tools」），不只做 tool

**Transport**（`CLAUDE.md:L103`）：stdio（默认）+ SSE（deprecated）+ Streamable HTTP。

### mcpServers JSON 客户端配置

**Claude Desktop 标准**（`README.md:L82-L91`）：
```json
{
  "mcpServers": {
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    }
  }
}
```

**Windows 包装**（`README.md:L93-L103`）：`command: "cmd"`, `args: ["/c", "npx", ...]` 防止 npx 直接调失败。

**env 注入**（`README.md:L118-L125`，如 github server）：通过 `env` 字段传 token（`GITHUB_PERSONAL_ACCESS_TOKEN`），**不**走 stdin 传 secrets。

**filesystem 必传路径**（`README.md:L113`）：`/path/to/allowed/files` 作为**最后一个 arg**——server 启动时限制允许目录。
