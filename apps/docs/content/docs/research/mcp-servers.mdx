# mcp-servers

## 一句话定位

> Anthropic 官方维护的 MCP **参考实现集合**——7 个 small server（4 TS + 3 Py），演示 Tools/Resources/Prompts/Roots 用法并作 SDK 样板；生产集成请去 MCP Registry。

## 关键 takeaway

1. **reference，不是产品目录**——README 显式「not production-ready」(skill_collection:L9)；完整 server 列表走 MCP Registry (skill_collection:L6)。
2. **monorepo 装 7 个 server**（4 TS + 3 Py），各自独立 npm/PyPI 包：everything / fetch / filesystem / git / memory / sequentialthinking / time (skill_collection:L29-L35)。
3. **13 个老 server 已归档**到 servers-archived（含 github、postgres、gdrive、puppeteer、redis、sentry、slack、sqlite、brave-search 等）(skill_collection:L41-L53)。
4. **零 clone 跑**：`npx -y @modelcontextprotocol/server-memory` 或 `uvx mcp-server-git` (skill_collection:L61-L75)。
5. **真用靠 MCP client config**——`mcpServers` 写 `command`+`args`+`env`；Windows 用 `cmd /c` 包 npx (skill_collection:L80-L134)。
6. **CI/SDK 范式**——TS：`npm workspaces` + `tsc` + `vitest`（Node 22）；Py：`uv` + `hatchling` + `pyright` + `ruff`（Py ≥ 3.10）(skill_collection:CLAUDE.md)。
7. **Tool annotations**——filesystem server 在 tool 上声明 `readOnlyHint` / `idempotentHint` / `destructiveHint` (skill_collection:filesystem/README.md:L189-L205)。
8. **License 双轨**：MIT（旧）+ Apache-2.0（新），文档 CC-BY-4.0 (skill_collection:L154, LICENSE:L1-L5)。
