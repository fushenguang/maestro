# Claude Code 总览（官方文档）

## 一句话定位

> Anthropic 官方的 agentic 编码工具入口，覆盖安装、能力矩阵与多端协同。

## 关键 takeaway

1. Claude Code 是 agentic 编码工具，读代码库、改文件、跑命令并接入开发工具链（终端、IDE、桌面、浏览器）。锚点 `docs:L1`。
2. 五大安装/运行环境：Terminal CLI、VS Code、Desktop、Web（claude.ai/code）、JetBrains；CLI 与 VS Code 额外支持第三方 provider。锚点 `docs:L1`。
3. 能力分 6 块：自动化杂活、构建功能/修 bug、git 提交与 PR、MCP 工具连接、CLAUDE.md/skills/hooks 自定义、Agent SDK 与并行 agent。锚点 `docs:L1`。
4. 通过 MCP 接入 Google Drive/Jira/Slack 等外部数据源；CLAUDE.md 作为项目级持久指令，hooks 在动作前后跑 shell。锚点 `docs:L1`。
5. 所有 surface 共享同一 Claude Code 引擎，CLAUDE.md/设置/MCP 跨端通用，并可与 CI/CD、Slack Channels、GitHub Actions 等集成。锚点 `docs:L1`。

## 与 Claude Code 教程的关联

这是教程的「全景地图」源：可作为入门章节导览，引出安装方式、CLAUDE.md 配置、MCP 接入、Agent SDK 等子主题的深读链接。对「快速上手」与「多端协作」两节直接可用。

## Brief（梳理）

### Claude Code 定位：agentic 而非 chat

- **核心定位**：agentic 编码工具；自主读代码、改文件、跑命令、接开发工具链
- **设计哲学**：5 端（CLI / VS Code / Desktop / Web / JetBrains）共享同一引擎；CLAUDE.md / settings / MCP 跨端通用
- **与 chat 工具本质区别**：被动 Q&A vs 主动「行动代理」——这是教程应强调的第一个概念

### 5 端选择决策表

| Surface | 适用场景 | 第三方 provider |
|---|---|---|
| CLI | 服务器 / SSH / CI/CD | ✓ |
| VS Code | 日常开发（已用 VS Code 团队）| ✓ |
| Desktop | 不想配 IDE 集成 | — |
| Web（claude.ai/code）| 远程 PR review / 无本地环境 | — |
| JetBrains | 已用 JB 工具链 | — |

- **起步推荐**：CLI（最低门槛）+ 任意 IDE 端
- **CI/CD 集成**：CLI 是事实标准（GH Actions / CircleCI）
- **Slack / Web**：可作为「非开发场景」（PR review / 客服 / SRE 报警）

### 6 大能力 = 教程 6 大章节

1. 自动化杂活（grep / sed / test / lint / format）
2. 构建功能 / 修 bug（从 issue / PR 到多文件实现）
3. Git 提交与 PR（commit message + PR body + CI 修复）
4. MCP 工具连接（Google Drive / Jira / Slack / DB / 自定义 server）
5. CLAUDE.md / skills / hooks 自定义（项目级指令 + 可复用流程 + 动作拦截）
6. Agent SDK + 并行 agent（自定义 agent 编排）

> **关键洞察**：6 大能力不是 6 个 feature，是教程**骨架**——每块对应 5-15 节内容

### MCP 协议：连接外部世界

- Anthropic 推出的开放标准，连接 AI 工具与外部数据源
- Claude Code 是 MCP 旗舰实现（详读 `anth-mcp`）
- 教程单独成章

### CLAUDE.md + skills + hooks 三件套

| 组件 | 角色 |
|---|---|
| CLAUDE.md | 项目指南（每个 session 读）|
| skills | 可复用工作流 |
| hooks | 动作拦截（pre / post tool use）|

- 三者互补：指南 + 流程 + 拦截器
- 详读 `anth-memory`（CLAUDE.md 4 级加载）+ `anth-hooks`（hooks 配置）

### Agent SDK：拆 Claude Code 当库

- Agent SDK = 库 + 你的进程（不是 hosted 服务）
- 用 Claude Code 的 tool / capability 在自己代码里造 agent
- 2026-06-15 起新 credit 配额（详读 `anth-agent-sdk`）
- 教程决策点：何时用 SDK vs Claude Code 本身
