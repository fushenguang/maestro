# aider

## 一句话定位

> 开源 terminal 配对编程 agent（"AI pair programming in your terminal"），三大支柱：git-first + 多 edit format + pluggable LLM，强调人在回路 + 自动 commit 的可审计工作流。 (methodology:L1)

## 关键 takeaway

1. **Pair programmer 而非 autonomous agent**。每轮 diff 呈现给用户 review，命令式 chat 循环。 (methodology:L2)
2. **git-first 是底座**。每次 edit 后自动 commit，`/undo` 一键回退——LLM 行为落到 git 审计链。 (methodology:L3)
3. **多 edit format 是杀手特性**。`aider/coders/` 内置 14+ 种（whole / diff / udiff / editor-diff / editblock / patch / architect / ask / help / context），按模型自动挑最稳 prompt。 (methodology:L4)
4. **pluggable LLM**。后端 `litellm`，OpenAI / Anthropic / DeepSeek / Gemini / 本地 Ollama 通吃，**不绑单家**。 (methodology:L5)
5. **Repo-map = RAG-for-codebase**。扫描全 repo 抽 token 化调用图，模型在大项目里也能定位要改的文件——后继 RAG-coding 工具的共同祖先。 (methodology:L6)
6. **chat modes 显式分层**。`/code`（改）/ `/ask`（讨论）/ `/architect`（plan + editor 双模型）/ `/help`，CC 的 sub-agent / plan mode 借鉴这套。 (methodology:L7)
7. **OSS + 零 telemetry + Apache 2.0**。6.8M PyPI 安装、15B tokens/周，自报 "88% 新代码由 Aider 自己写" (Singularity badge)。 (methodology:L8)

## 与 Claude Code 的对比

- **哲学**：aider = pair programmer 在回路；CC = agent 派任务自跑。
- **LLM**：aider = pluggable (litellm) 10+ 厂商；CC = Anthropic-only。
- **Edit 表达**：aider = 14+ format 模型自选；CC = unified tool-use 协议。
- **Git**：aider = 每次 edit 自动 commit；CC = 手动或 hook。
- **可观测**：aider = 全 commit 链 + `/undo`；CC = task 子 agent + transcript。
- **形态/商业**：aider = Python CLI + Apache 2.0 零遥测；CC = Node CLI + 闭源。

CC 借鉴：plan/architect 双模型、ask/code 切换、git-aware undo。Aider 是 ML 团队跑 LLM benchmark 的「参考线」（polyglot bench），不会消失。

## 与 Claude Code 教程的关联

- **同源**：Aider `architect` = CC plan-then-execute；`/ask` = CC plan permission mode；`/undo` = CC checkpoint + revert。
- **互补**：Aider 跑 GPT-4o/DeepSeek 横向对比 + benchmark（不烧 Anthropic 配额）；CC 跑 Anthropic-native 长任务、sub-agent fan-out。
- **盲点**：Aider 没 CC 的 hooks / MCP / sub-agent / permission 系统——「企业级 agent 平台」是 CC 赛道，aider 是「个人开发者最强 IDE 增强」赛道。
