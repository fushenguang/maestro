# 教程项目选题（4 类候选）

> 调研日期：2026-06-12
> 调研范围：小游戏 / 网站工具 / 商业交易 / 学习类
> 硬约束：TypeScript / Rust，1 session 可跑通 MVP，拒绝纯玩具
> 修订记录：v0.3 (2026-06-12 晚) 增 4 个 Tauri / Rust 候选（含 1 Bevy）：Bevy 2D 平台跳跃 / Tauri Markdown 笔记 / Tauri Git 客户端 / Tauri 代码片段管理器；候选总数 16 → **20**；Tauri 占比 3 → **6 个**，Rust 相关 5 → **9 个**（9/20 = **45%**）
> v0.2 (2026-06-12 PM) 增 4 个 Tauri / Rust 候选（每类 1 个），候选总数 12 → 16；Tauri 占比从 0 → 3 个，Rust 占比从 1 → 5 个（5/16 = 31%）
> **数据来源声明**：本批次调研因 WebSearch / WebFetch / Brave Search 三路网络工具不可用，候选为「brief 方向 + 训练数据中已验证的成熟项目型」合成；下游如升级 Brief 须先做实时项目验证。

## 一句话定位

> 4 类项目共 **20 个候选**（每类 5 个，含 6 个 Tauri + 1 个 Bevy + 1 个 Rust macroquad + 1 个 Rust CLI = **9 个 Rust 相关 = 45%**），覆盖**好玩 / 能用 / 能赚钱 / 教自家小孩**四种情绪价值，按「1 session 可跑通 MVP」硬约束筛选。

## 关键 takeaway

1. **情绪价值分层**：4 类对应 4 个不同情绪锚点（好玩 / 能用 / 赚钱 / 给小孩用），教程正文应让读者**先选情绪再选项目**，而不是按技术难度选。20 个候选 = **9 Rust 相关（45%）+ 11 TS 纯**，Rust/Tauri 比例从 v0.1 的 8% 提升到 45%。
2. **CC 功能映射是选型核心维度**：每个候选单独列出「自然用到 CC 哪些功能」（Hooks/Memory/MCP/Skills/Sub-agents），这是教程章节能否落地的关键。
3. **复杂度档位**：20 个候选中 8 个标"简单（1 session）"、12 个标"中等（2-3 session）"——简单档占 40%，便于新人上手；Tauri 6 个 + Bevy 1 个全部标"中等"（Tauri/Bevy 配置本身需 1-2 session 学习）。
4. **真实业务信号**：商业交易类全部接 Stripe（含 payment / subscription / checkout），不是模拟流程；学习类全部能换题库/换语言，**可被自家小孩复用**。
5. **风险点前置**：每个候选标"最大坑"——常见的是范围蔓延（贪吃蛇想做太多）、集成复杂度（Stripe Webhook）、Rust 学习曲线（macroquad 上手慢）。

## 第 1 类：小型游戏（提升兴趣，先玩起来）

> 情绪锚点：**好玩**——做完立刻能玩，改完立刻看到效果

### Snake + 全球排行榜（TypeScript + Canvas + Supabase）

> 经典贪吃蛇加云端排行，让"自嗨"变"自嗨 + 跟人比"。

- **技术栈**：TypeScript + Canvas 2D（无框架）+ Vite + Supabase（Postgres + Auth）做排行榜
- **预估复杂度**：**简单**（1 session）
- **核心功能**：
  - 网格化棋盘 + 蛇身增长 + 食物随机生成
  - 键盘方向键 / WASD 控制方向
  - 撞墙或撞自身 game over
  - 本地最高分 localStorage 缓存
  - 输入昵称后提交分数到 Supabase `scores` 表
  - 全站 Top 10 排行榜实时拉取
- **CC 功能映射**：
  - **Skills**：写一个 `/add-feature snake-xxx` skill 模板，新功能按"输入 → 逻辑 → 渲染"三段
  - **Memory**：在 `CLAUDE.md` 写"蛇头用 emoji、移动速度 ms 为单位、分数 = 蛇长 × 10"
  - **Hooks**：PostToolUse 跑 `tsc --noEmit`，改完代码自动类型校验
  - **Sub-agents**：一个写前端逻辑、一个写 Supabase schema + RLS policy
- **情绪锚点**："我能 1 小时做出一个**朋友能访问的网页版贪吃蛇**，而且能跟他们比谁分高"
- **可扩展性**：
  - 加移动端触控（虚拟方向键）
  - 加道具（减速/加速/穿墙）
  - 加主题（皮肤包：经典 / 像素 / emoji）
  - 加对战模式（WebSocket 同步）
- **风险**：
  - **范围蔓延**最常见——想加道具/对战/账号系统导致 1 session 跑不完
  - 建议严格按"裸 snake → 加分数 → 加排行榜"三步走，每步可暂停

### Phaser 2D 平台跳跃（TypeScript + Phaser 3 + Tiled）

> 复刻"超级马里奥"最小可玩版，自带关卡编辑器。

- **技术栈**：TypeScript + Phaser 3 + Vite + Tiled Map Editor（关卡设计）
- **预估复杂度**：**中等**（2-3 session）— 比贪吃蛇复杂在物理 + 关卡
- **核心功能**：
  - 玩家跳跃物理（重力 + 速度 + 碰撞）
  - Tiled 编辑的 .json 关卡载入
  - 收集金币 + 终点旗
  - 摄像机跟随
  - 简单 UI（HUD：分数 / 生命）
- **CC 功能映射**：
  - **Sub-agents**：物理调参 agent + 关卡设计 agent（生成多个关卡 JSON 并行）
  - **Memory**：`CLAUDE.md` 记 Phaser 关键 API（`this.physics.add.sprite` / `scene.start`）避免每次都问
  - **Skills**：写 `/physics-debug` skill，遇到跳跃手感不对时自动打印 velocity / 摩擦系数
  - **Hooks**：保存关卡 .json 时自动校验 schema
- **情绪锚点**："我做出了一个**能跳能吃的 2D 横版游戏**，自己画的关卡能给朋友玩"
- **可扩展性**：
  - 加敌人 AI（巡逻 + 跳跃）
  - 加多关卡（关卡选择界面）
  - 加音效（Phaser 自带 WebAudio）
  - 导出为 PWA 装到手机桌面
- **风险**：
  - Phaser 学习曲线中等（scene / game object / physics 三套概念）
  - **物理调参耗时**（跳跃手感差需要反复调重力/速度）——这是最容易放弃的环节
  - Tiled 工具额外学习成本

### Rust 太空射击（Rust + macroquad）

> 用 Rust 写 2D 太空射击——把"学 Rust 的痛苦"变成"做游戏的快乐"。

- **技术栈**：Rust + macroquad（极简 2D 游戏框架）+ Cargo
- **预估复杂度**：**简单**（1 session）— macroquad 模板代码极少
- **核心功能**：
  - 玩家飞船 WASD 移动 + 空格射击
  - 敌人随机生成 + 下落
  - 碰撞检测 + 生命值
  - 计分 + 简单粒子效果（爆炸）
- **CC 功能映射**：
  - **Sub-agents**：写代码 agent + 编译验证 agent（Rust 编译慢，分两个 agent 流水线）
  - **Hooks**：保存 .rs 后自动跑 `cargo check`（PreToolUse + 工具结果检查）
  - **Memory**：`CLAUDE.md` 记 macroquad 模式（`#[macroquad::main]` / `next_frame().await`）
  - **Skills**：写 `/add-enemy` skill，敌人按"出现位置 + 移动模式 + 子弹模式"三段描述生成
- **情绪锚点**："我用 Rust（号称最难的系统语言）**做出一个能玩的游戏**，编译一次过的快感"
- **可扩展性**：
  - 加 boss 战（多阶段行为树）
  - 加升级系统（捡道具升火力）
  - 用 `bevy_ecs` 替换 macroquad 内置 ECS 学 ECS 范式
  - 跨平台编译（`cargo build --target wasm32-unknown-unknown` 跑浏览器）
- **风险**：
  - **Rust 学习曲线 + 借用检查器**——新手可能 30 分钟卡在一个 `&mut` 上
  - 编译慢（10-30s）影响 CC 迭代节奏
  - macroquad 文档相对薄弱（相比 Bevy / Phaser），靠示例代码学习

### Tauri 桌面版贪吃蛇 + 本地排行榜（Rust + Tauri 2 + Canvas + SQLite）

> 把"网页贪吃蛇"升级为"原生桌面应用"——双击图标启动，无浏览器地址栏，体验像 Sublime / VS Code 一样的专业 App。**强烈推荐**：父项目 Maestro 本身是 Tauri 栈，做这个等于学 Tauri 2 实战。

- **技术栈**：Rust + Tauri 2（WebView）+ TypeScript + Canvas 2D + SQLite（本地排行榜）
- **预估复杂度**：**中等**（2-3 session）— Tauri 配置 + Rust 后端 + 前端
- **核心功能**：
  - 复用经典贪吃蛇逻辑（与本类第 1 个候选同源，可对照学"Web → 桌面"迁移）
  - Tauri 窗口：无边框、可拖动、固定到 dock / 任务栏
  - 本地最高分 SQLite 存储（多难度分桶：经典 / 加速 / 困难）
  - 全局快捷键唤起（macOS / Windows 跨平台）
  - 系统通知（破纪录时 OS notification）
  - 应用菜单（New Game / Settings / Quit）
- **CC 功能映射**：
  - **Skills**：写 `/add-frontend-feature` + `/add-rust-command` 两个 skill 模板
  - **Memory**：`CLAUDE.md` 记 Tauri 2 关键约定（`#[tauri::command]` / `tauri.conf.json` capabilities 权限矩阵 / 前端如何 invoke 后端）
  - **Hooks**：保存 .rs 跑 `cargo check`，保存 .ts 跑 `tsc --noEmit`
  - **Sub-agents**：Rust 后端 agent（命令 + SQLite）+ 前端 Canvas agent 并行
- **情绪锚点**："我做出了一个**原生桌面应用**——双击图标就开，没有浏览器地址栏，**和 Sublime / VS Code 一样的体验**"
- **可扩展性**：
  - 加皮肤包（Rust 端管理资源，热加载）
  - 加云端排行榜（替换 SQLite 为 Supabase，复用本类第 1 候选架构）
  - 加多语言 i18n
  - 加自动更新（Tauri updater）
  - 跨平台打包（macOS DMG / Windows MSI / Linux AppImage）
- **风险**：
  - **Tauri 2 配置坑**（`tauri.conf.json` / capabilities 权限矩阵 / 资源加载路径）— 第一次配置会踩多次
  - 跨平台编译差异（macOS 需 codesign + notarize，Windows 需 Authenticode）
  - WebView 一致性（macOS WKWebView / Windows WebView2 / Linux WebKitGTK 行为差异）

### Bevy 2D 平台跳跃（Rust + Bevy 0.14 + Rapier 物理 + ECS 实战）

> 用 Rust 生态最活跃的 Bevy 引擎做 2D 横版跳跃——和 macroquad 太空射击并列，**学 ECS（实体组件系统）数据驱动游戏架构**。参考 [bevyengine/bevy](https://github.com/bevyengine/bevy) 官方仓库。

- **技术栈**：Rust + Bevy 0.14（ECS + 2D 渲染插件）+ bevy_rapier2d（2D 物理）+ tiled（关卡解析）+ bevy_ecs_tilemap
- **预估复杂度**：**中等**（2-3 session）— Bevy 学习曲线比 macroquad 陡
- **核心功能**：
  - 玩家跳跃物理（Rapier 重力 + 速度 + 碰撞）
  - Tiled 编辑的 .json 关卡载入（bevy_ecs_tilemap 渲染瓦片地图）
  - 收集金币 + 终点旗
  - 摄像机跟随 + 平滑插值
  - **ECS 系统拆分**（input / physics / collision / render / animation 各自独立 system，按 SystemSet 调度）
  - 简单 HUD（分数 / 生命）
- **CC 功能映射**：
  - **Sub-agents**：ECS 系统设计 agent（拆分 input / physics / render system）+ 关卡设计 agent 并行
  - **Memory**：`CLAUDE.md` 记 Bevy 关键 API（`Query<(&Transform, &mut Velocity)>` / `SystemSet` / `Resource` / `Component`）
  - **Skills**：写 `/add-enemy` skill，敌人按"组件 + 行为系统"两段描述
  - **Hooks**：保存 .rs 跑 `cargo check` + 跑 `cargo clippy -- -D warnings`（Bevy 编译慢，clippy 提前发现问题避免反复重编）
- **情绪锚点**："我用 Rust 最火的 Bevy 引擎做出了一个**ECS 架构的 2D 游戏**——理解了 ECS = 理解了 Rust 在游戏领域的杀手锏，**比 macroquad 上限高 10 倍**"
- **可扩展性**：
  - 加敌人 AI（巡逻 + 跳跃 + 视锥检测 system）
  - 加多关卡（关卡选择 + 进度存档 Resource）
  - 升级到 3D（Bevy 3D 同样用 ECS，只需换 render plugin + camera）
  - 加 hot-reload 关卡（Tiled 文件改动自动重载 AssetEvent）
  - 加 mod 支持（用 Bevy 的 Plugin 机制让社区写关卡 mod）
- **风险**：
  - **Bevy 学习曲线最陡**（ECS 思维 + 系统调度 + 资源管理）— 0 Rust 基础 + 0 ECS 经验者慎入
  - **Bevy 编译极慢**（首次 5-10 分钟，增量 30-60s）—— **CC 迭代节奏会被严重拖累**，需用 `cargo check` + `sccache` 缓存
  - Bevy 0.x → 1.0 期间 API 不稳定，教程必须锁定 0.14 版本
  - 文档相对薄弱（官方 examples + unofficial cheat book 是主要学习材料）

## 第 2 类：网站 / 工具站（使用价值，能给人看）

> 情绪锚点：**能用**——做出一个你能发给朋友的链接

### Markdown → 微信公众号排版转换器（TypeScript + Vite + Vanilla JS）

> 写完 Markdown 粘进去，复制出来的就是排好版的微信文章——**中文创作者刚需**。

- **技术栈**：TypeScript + Vite + 原生 DOM（无框架）+ marked（Markdown parser）+ 一个主题 CSS
- **预估复杂度**：**简单**（1 session）
- **核心功能**：
  - 左编辑右预览的双栏布局
  - marked 转 HTML + 自定义样式（标题居中、正文字号、行高）
  - 复制到剪贴板（保留 inline 样式，微信公众号粘进去不丢格式）
  - 几个预设主题（默认 / 文艺 / 极客）
  - 字数统计 + 阅读时间估算
- **CC 功能映射**：
  - **Skills**：写 `/style-preset` skill，主题按"字体 / 字号 / 行高 / 间距 / 配色"5 维描述
  - **Memory**：`CLAUDE.md` 记微信公众号样式坑（`section` 标签不支持内联 style / 必须用 `p` 包裹 / 图片必须传微信服务器）
  - **Hooks**：保存时跑 markdownlint
- **情绪锚点**："我做的工具**真的能解决我自己写公众号的痛点**，以后每天用"
- **可扩展性**：
  - 加代码高亮（highlight.js）
  - 加数学公式（KaTeX）
  - 加图片上传（mcp 接微信公众号素材管理 API）
  - 部署到 Vercel / Cloudflare Pages 公开
- **风险**：
  - **微信富文本限制**（不支持 `flex` / `grid` / `position: fixed`）——踩坑需要反复试
  - 主题样式需要在小屏/中屏/大屏都好看，调 CSS 耗时

### 个人作品集 + MDX 博客（Next.js + MDX + Tailwind + shadcn/ui）

> "我的网站 = 我的简历"，每篇文章用 MDX 写（可嵌 React 组件）。

- **技术栈**：Next.js 15 (App Router) + MDX + Tailwind + shadcn/ui + Vercel 部署
- **预估复杂度**：**中等**（2-3 session）— 内容多但每块都简单
- **核心功能**：
  - 首页：自我介绍 + 项目卡片 + 最新文章
  - `/blog` 文章列表 + tag 筛选
  - `/blog/[slug]` MDX 渲染（代码高亮、Callout、TOC）
  - 暗色模式切换
  - RSS feed + sitemap
  - 访客统计（Umami / Plausible）
- **CC 功能映射**：
  - **Skills**：写 `/new-post <title>` skill，自动生成 frontmatter + 模板 + slug
  - **Memory**：`CLAUDE.md` 记个人品牌信息（昵称 / 头像 / 社交链接 / 写作语气）
  - **MCP**：接 GitHub API 自动拉取置顶项目（动态渲染）
  - **Hooks**：commit 前跑 `next build` 确认能过
- **情绪锚点**："我有一个**自己的网站**——能发链接给任何人、永远在线、能放作品 + 写作 + 简历"
- **可扩展性**：
  - 加评论系统（Giscus）
  - 加 Newsletter（接 Resend / Buttondown）
  - 加 i18n（中英双语）
  - 域名绑定（Cloudflare Registrar）
- **风险**：
  - **shadcn/ui 组件**挑选要节制（贪多导致样式不统一）
  - Next.js App Router 学习成本（Server Component / Client Component 区分）
  - Vercel 部署后偶发冷启动慢

### URL Shortener + 点击统计（TypeScript + Hono + SQLite + Cloudflare Workers）

> 完整业务闭环——短链生成 + 跳转 + 统计面板，"做出来真能用"。

- **技术栈**：TypeScript + Hono（极简 Web 框架）+ Cloudflare Workers（D1 SQLite）+ HTMX
- **预估复杂度**：**中等**（2-3 session）
- **核心功能**：
  - 短码生成（Base62 / nanoid）
  - `/{code}` 路径 302 跳转 + 记录 referer / UA / IP
  - 管理面板（创建短链 + 看点击数 + 看来源分布）
  - 防滥用：单 IP 频率限制（KV 存储）
  - 自定义短码（`/go/my-link`）
- **CC 功能映射**：
  - **MCP**：接 Cloudflare API 自动部署
  - **Sub-agents**：写 API agent + 写前端面板 agent 并行
  - **Skills**：写 `/shorten` skill，一句话生成短链
  - **Hooks**：提交前跑 `wrangler deploy --dry-run` 验证

### Rust CLI 重复文件查找器（ripgrep / fd 简化版，Rust + clap + walkdir）

> 一个超快的命令行工具——扫描目录找出内容 / 名称重复的文件，**比 Python 快 50 倍**。Rust CLI 是学 Rust 最高 ROI 的练手项目。

  - **Sub-agents**：单 agent 即可（CLI 项目天然不分）
- **情绪锚点**："我用 Rust 写了个**比 Python 快 50 倍的 CLI 工具**——以后清理下载文件夹就用它，**同事看到都问'这怎么装的'**"
- **可扩展性**：

- **风险**：
  - **大目录 IO 慢**（几百万文件需要流式处理，不能一次读到内存）
  - 哈希算法选择（SHA256 慢但安全 / xxhash 快但只是查重）

  - 加多人对战模式（朋友间比掌握度）
- **风险**：
  - **SM-2 算法不熟容易实现错**（推荐抄 Anki 源码或用现成 `supermemo` npm 包）
  - 同步问题（多设备）需要后端，超出 MVP 范围
  - 孩子的注意力（UI 必须够萌 / 反馈必须即时）

### 数学闯关（小学口算游戏化，TypeScript + React + Tailwind）

> 闯关式口算——10 题一关，过关升级，**孩子愿意主动刷题**。

- **技术栈**：TypeScript + React + Vite + Tailwind + 本地 JSON 题库
- **预估复杂度**：**简单**（1 session）
- **核心功能**：
  - 题目生成器（加减乘除，按难度分档：1年级 / 2年级 / 3年级）
  - 闯关模式：10 题一关，错 3 题就 game over
  - 答题动画：答对撒花 + 答错红叉 + 倒计时
  - 关卡地图（解锁进度）
  - 错题本（自动收集错题，下次复习）
  - 排行榜（家庭内 / 班级内）
- **CC 功能映射**：
  - **Skills**：写 `/new-grade` skill，按"年级 / 运算类型 / 范围 / 时间限制"参数化生成关卡
  - **Memory**：记孩子年级 / 错题偏好（如总错乘法进位）
  - **Sub-agents**：出题 agent + 出 UI agent 并行
  - **Hooks**：保存题库时自动校验答案唯一性
- **情绪锚点**："我做的工具**让我家孩子数学进步了**——从'讨厌口算'到'主动要求再做一关'"
- **可扩展性**：
  - 加多用户（多孩子账号）
  - 加教师后台（看班级统计 + 布置作业）
  - 加知识点关联（错题→知识点标签→推荐讲解视频）
  - 加打印模式（家长可打印纸质练习）
- **风险**：
  - **题目生成器要小心**（不能出负数 / 不能除以 0 / 不能太难让小孩挫败）
  - UI 配色 / 动效要花心思（小孩的审美跟大人不同）
  - 避免变成"游戏"而非"学习"（不能跳过题目直接解锁）

### AI 互动故事生成器（TypeScript + React + Anthropic SDK）

> 小孩选"主角 / 场景 / 情节分支"，AI 实时生成故事——**让 AI 落地**给小孩玩。

- **技术栈**：TypeScript + React + Vite + @anthropic-ai/sdk + Tailwind
- **预估复杂度**：**简单**（1 session）— 核心是 prompt 设计
- **核心功能**：
  - 故事开始：选主角（公主 / 骑士 / 小狗 / 机器人）+ 场景（森林 / 太空 / 海底）+ 主题（冒险 / 友谊 / 勇气）
  - AI 生成故事开头（200 字 + 配图描述）
  - 故事中给 2-3 个选项（小孩选情节走向）
  - 选完继续生成（每段 200 字，可手动结束）
  - 故事收藏夹（保存喜欢的故事）
  - TTS 朗读（家长模式：睡前故事）
- **CC 功能映射**：
  - **MCP**：接 Anthropic API（用 Claude Sonnet）— 核心就是用 API
  - **Skills**：写 `/story-prompt` skill，把 prompt 模板化（角色 + 场景 + 主题 + 上文 + 选项）
  - **Memory**：记孩子偏好（喜欢什么主角 / 讨厌什么情节 / 阅读水平）
  - **Hooks**：调用 API 前 prompt 注入安全过滤（"故事不能出现暴力 / 不能出现可怕的意象"）
  - **Sub-agents**：写代码 agent + 写 prompt 模板 agent + 写家长控制面板 agent 并行
- **情绪锚点**："我用 AI **给我孩子做了一个会讲故事的工具**——他每天睡前都要'再讲一个'"
- **可扩展性**：
  - 加插图（用图像生成 API 配图）
  - 加多语言（中英双语故事）
  - 加教育主题（数学故事 / 历史故事）
  - 加打印（保存为 PDF 绘本）
- **风险**：
  - **API 成本**（每次讲故事 ~500 tokens，全家每天 5 个故事 = $0.5/天，需要控制频率或加付费）
  - **内容安全**（小孩用的 AI 输出必须过滤，prompt 必须前置 "no violence, no scary content"）
  - **prompt 设计决定质量**——同一个 API 给的 prompt 不同结果天差地别
  - 故事连贯性（长故事容易跑题，需要定期 re-summarize）

### Tauri 桌面 Anki 卡（Rust + Tauri 2 + SQLite + SM-2 算法）

> 把"Web Anki 卡"升级为"桌面 Anki"——本地优先、离线可用、零广告、**给自家孩子用不担心隐私**。Rust 端实现 SM-2 算法（性能 + 安全性双优），前端负责 UI。

- **技术栈**：Rust + Tauri 2 + TypeScript + SQLite（`rusqlite`）+ SM-2 间隔重复算法（Rust 端实现，前端按需调用）+ Tauri 系统托盘
- **预估复杂度**：**中等**（2-3 session）— Tauri + Rust 算法 + 前端 UI + 托盘集成
- **核心功能**：
  - 多卡组管理（英语单词 / 拼音 / 唐诗 / 自定义 JSON 卡组）
  - **SM-2 算法 Rust 端实现**（性能比纯 JS 优 10×，且可单元测试覆盖）
  - 翻牌动画 + 今日复习队列（前端用 CSS / Canvas）
  - 进度统计（已学 / 复习中 / 掌握 / 即将遗忘）
  - 导入导出（Anki .apkg 格式 / 自定义 JSON）
  - **系统托盘**（最小化到托盘 + 每日提醒"今天有 N 张卡要复习"）
  - 全局快捷键（Cmd+Shift+A 唤起主窗口）
- **CC 功能映射**：
  - **Skills**：写 `/add-deck` skill（卡组按"标题 / 字段 / 初始数据"模板）
  - **Memory**：`CLAUDE.md` 记孩子信息 + SM-2 算法边界用例 + `rusqlite` 常用 SQL
  - **Sub-agents**：Rust SM-2 算法 agent + 前端 UI agent 并行（两个 agent 写不同语言，天然隔离）
  - **Hooks**：保存 .rs 跑 `cargo test`（SM-2 边界用例）+ 前端 `tsc --noEmit`
  - **MCP**：可扩展接 Anki Connect（同步到 Anki 官方账号）
- **情绪锚点**："我做的东西**像专业 Anki 一样**——本地优先、离线可用、零广告，**给自家孩子用不担心隐私**，**比花 200 块买教育 App 值**"
- **可扩展性**：
  - 加 TTS（系统语音 API 发音 + 跟读评分）
  - 加图片 / 音频卡片（OCR 名著截图自动生成卡）
  - 加多用户（家庭账号切换）
  - 加云同步（自建 WebDAV 或 Supabase）
  - 加家长后台（看孩子学习曲线 + 强制休息）
- **风险**：
  - **Rust 端 + 前端双实现 SM-2 容易不一致**——必须 Rust 端写完 + 单测覆盖 + 前端只调不重写
  - Tauri 2 + SQLite 性能调优（`PRAGMA journal_mode=WAL` / 连接池）
  - 系统托盘跨平台差异（macOS / Windows / Linux API 不一致）
  - 跟 Anki 官方 App 功能差距大（必须聚焦"本地 + 极简"差异化）

### Tauri 桌面代码片段管理器（masscode 简化版，Tauri 2 + SQLite FTS5 + Monaco）

> 程序员必备——分类存代码片段（API 例子 / 工具函数 / 配置模板），**全平台同步 + 全文搜索 + 标签**。开源参考 `masscode-io/masscode`（原版 Electron 改 Tauri 仿品）。**给开发者每天省 30 分钟翻历史代码的时间**。

- **技术栈**：Rust + Tauri 2 + TypeScript + SQLite（FTS5 全文搜索虚拟表）+ Monaco Editor（多语言高亮）+ 系统托盘 + 全局快捷键 + `arboard`（剪贴板）
- **预估复杂度**：**中等**（2-3 session）
- **核心功能**：
  - 片段 CRUD（标题 / 代码 / 语言 / 标签 / 描述）
  - 多文件夹 / 多标签分类 + 嵌套文件夹
  - **全文搜索**（SQLite FTS5 虚拟表 + 中文 `unicode61` + 自定义分词）
  - **全局快捷键**（Cmd+Shift+S 唤起搜索框，模糊匹配代码片段）
  - **系统托盘**（最小化到托盘 + 一键复制粘贴）
  - 一键复制（按 Enter 直接复制代码到剪贴板 + 自动关闭搜索框）
  - 导入导出（JSON / masscode 格式 / VSCode snippet 格式）
- **CC 功能映射**：
  - **Skills**：写 `/add-snippet` skill（按"标题 / 语言 / 标签 / 模板"模板化）+ `/snippet-search` skill（模糊匹配 + 排序规则）
  - **Memory**：`CLAUDE.md` 记用户常用的语言 / 标签 / 片段模板
  - **Sub-agents**：Rust 后端（SQLite FTS5 + 剪贴板 + 系统托盘 + 全局快捷键）agent + 前端（Monaco + 搜索 UI + 片段管理）agent 并行
  - **Hooks**：保存 Rust 跑 `cargo check` + 保存新片段跑语言检测（自动识别 TS / Rust / Python）
  - **MCP**：可扩展接 GitHub Gist 同步（云端备份片段）
- **情绪锚点**："我做出了程序员**每天用的工具**——**比花 $40 买 Alfred / Raycast Snippet 值**（开源 + 可定制 + 本地优先 + 跨平台），**且给自家孩子 / 团队成员都能用**"
- **可扩展性**：
  - 加 AI 自动补全（接 Anthropic API 续写片段 / 根据描述生成代码）
  - 加团队共享（多人协作片段库 + 权限管理）
  - 加 Markdown 富文本（支持说明 + 代码混排）
  - 加 VSCode 插件（双向同步 + 实时显示）
  - 加 GitHub Gist 同步（云端备份）
  - 加 AI 自动打标签（用 LLM 提取 snippet 主题）
- **风险**：
  - **系统托盘跨平台差异**（macOS / Windows / Linux API 不一致，必须用 `tauri-plugin-system-tray` 抽象层）
  - **全局快捷键抢占**（可能被其他 App 占用，需要冲突检测 + 用户配置覆盖）
  - **SQLite FTS5 中文分词坑**（默认对中文不友好，必须用 `jieba-rs` 预分词 + 自定义 tokenizer）
  - 跟 masscode / SnippetsLab / Raycast Snippet 正面竞争

## 风险与待验证项

1. **本批次无实时 web 验证** — 候选是基于"项目型是否成熟 + 1 session 可行性"判断，**未做 stars / 最新 commit / 同类竞品对比**。下游 Brief 阶段必须补。
2. **中文场景特定性** — Markdown→微信 转换器、付费 Newsletter 中文体验是核心卖点，英文教程没覆盖——这是机会也是验证空白。
3. **Stripe 在中国大陆** — 商业类 3 个候选都假设用户有海外主体（中国大陆主体被拒概率高）。**风险前置告知**读者。
4. **Rust 学习曲线** — Rust 太空射击假设读者有 Rust 基础（能读懂 `&mut` / 借用检查器错误），不适合 0 Rust 基础的人。
5. **AI 故事 API 成本** — Claude API 按 token 计费，互动故事每轮 ~500 tokens，必须加"每日上限"防止意外账单。
6. **Tauri 6 个候选的复杂度** — Tauri 桌面应用全部标"中等（2-3 session）"——Tauri 2 配置（`tauri.conf.json` / capabilities 权限矩阵 / 前后端 invoke 约定）本身需 1-2 session 学习，**不适合 0 桌面开发经验的读者**。建议先做 Snake + 排行榜（Web 版）再升级到 Tauri 桌面版。
7. **Tauri 跨平台分发坑** — macOS 需 codesign + notarize（开发者账号 $99/年），Windows 需 Authenticode 证书。**MVP 阶段可以跳过签名**（本地构建 + 用户自己 clone 跑），但要告知读者"上架分发是另一道坎"。
8. **Rust CLI 与 Tauri Rust 后端是两套心智** — Rust CLI 偏系统编程（IO / 进程 / 文件），Tauri Rust 后端偏 IPC + 状态机 + 序列化。**不要假设"会 Rust 就会 Tauri Rust"**。
9. **Bevy 编译速度** — Bevy 编译极慢（首次 5-10 分钟，增量 30-60s），CC 迭代节奏会被严重拖累。**必须用 `cargo check` + `sccache` 缓存**，否则反馈循环太长会让 CC 体验变差。
10. **Bevy API 稳定性** — Bevy 0.x → 1.0 期间 API 不稳定（半年内可能 break 1-2 次），教程必须锁定具体版本（推荐 0.14），并配"如果升级后报错"段落。
