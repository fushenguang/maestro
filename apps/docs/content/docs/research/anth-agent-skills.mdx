# Agent Skills（官方文档）

## 一句话定位

> Anthropic 官方对 Skills 的定义、仓库结构与 SKILL.md 写法权威说明。

## 关键 takeaway

1. Skills 是「指令 + 脚本 + 资源」的目录，Claude 按需动态加载以提升专门任务表现（`README.md:L6`）。
2. 每个 skill 自包含在一个文件夹里，核心是 `SKILL.md`（frontmatter + instructions）（`README.md:L18`）。
3. 最小 `SKILL.md` 只需 `name` 和 `description` 两个 frontmatter 字段（`README.md:L78-81`）。
4. 仓库分三类：`./skills` 示例、`./spec` 规范、`./template` 模板（`README.md:L27-29`）。
5. 接入途径：Claude Code plugin marketplace、Claude.ai 付费计划、Claude API Skills endpoint（`README.md:L33-61`）。

## 与 Claude Code 教程的关联

- Claude Code 教程的 `/plugin marketplace add anthropics/skills` 命令直接对应本文「Try in Claude Code」段落（`README.md:L36`）。

## Brief（梳理）

### 仓库三层结构（`anthropics/skills`）

| 目录 | 角色 | 备注 |
|---|---|---|
| `./skills` | **示例池**：Creative & Design / Development & Technical / Enterprise & Communication / Document Skills 四类 | docx/pdf/pptx/xlsx 4 个是 source-available（非 open source），驱动 Claude 文档能力 |
| `./spec` | **规范**：Agent Skills 标准的权威定义（agentskills.io）| 与 [agentskills.io](http://agentskills.io) 双源同步 |
| `./template` | **模板**：`template-skill` 提供可复制起点 | 见 `README.md:L64-L84` 完整 SKILL.md 骨架 |

- **仓库角色**：展示「skills 能做什么」+ 给出最低规范的 SKILL.md——**不**承担生产 skill 分发（docs 入口承担）。

### SKILL.md frontmatter 事实标准

**最小必填**（`README.md:L86-L88`）：仅 `name` + `description` 两字段。

**完整 schema**（从 anthropics/skills 仓库实际 + agentskills.io 规范推断）：

| 字段 | 必填 | 用途 |
|---|---|---|
| `name` | ✅ | 唯一标识（lowercase + hyphens）|
| `description` | ✅ | 「做什么 + 何时触发」完整描述（用于按需加载判定）|
| `allowed-tools` | ❓ | CC 中限制 skill 可调的工具（Anthropic 文档另有约束）|
| `model` | ❓ | 强制走特定模型（Opus 4 / Sonnet 4 / Haiku）|
| `version` | ❓ | 升级时给 agent 重新加载的钩子 |
| `metadata` | ❓ | 自定义键值（author / tags / 依赖）|

- **设计原则**：最小集 2 字段确保低门槛；可选字段让 power user 细控。
- **Body 结构**（`README.md:L72-L84` 模板）：`# My Skill Name` + 自由 Markdown（`Examples` / `Guidelines` 等 H2）。

### 接入途径对比

| 途径 | 入口 | 适用场景 |
|---|---|---|
| **Claude Code plugin marketplace** | `/plugin marketplace add anthropics/skills` → `Browse and install plugins` → `document-skills` / `example-skills`（`README.md:L34-L51`）| 本地开发环境 |
| **Claude.ai 付费计划** | 内置 `document-skills` + 支持上传自定义 skill（`README.md:L55-L57`）| 团队非技术用户 |
| **Claude API** | Skills endpoint + `Skills API Quickstart`（`README.md:L59-L61`）| 服务端集成 / CI |

- **互通点**：同一份 `SKILL.md` 在 3 端都能跑——schema 是跨端事实标准。
- **差异**：CC 通过 plugin marketplace 拉，Claude.ai 走 GUI，API 走 REST。
