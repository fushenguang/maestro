# Slash Commands（官方文档）

## 一句话定位

> 官方将原 Slash Commands 文档重写为 Skills 体系：自定义命令已并入 Skills（`.claude/commands/foo.md` 与 `.claude/skills/foo/SKILL.md` 等价），通过 SKILL.md YAML frontmatter 控制行为、可见性、工具权限、动态上下文与子代理执行。

## 关键 takeaway

1. **Custom commands = Skills（同名等价）**：`.claude/commands/deploy.md` 与 `.claude/skills/deploy/SKILL.md` 都创建 `/deploy`，二者作用一致；旧 `.claude/commands/` 文件仍可用但已被并入 Skills 体系。锚点 `docs:L1`。
2. **四级加载范围**：`enterprise > personal (~/.claude/skills/) > project (.claude/skills/) > plugin`（`plugin-name:skill-name` 命名空间）；冲突按层级解析。
3. **Frontmatter 16 字段**：`name`、`description`、`when_to_use`、`argument-hint`、`arguments`、`disable-model-invocation`（user-only）、`user-invocable`（Claude-only）、`allowed-tools` / `disallowed-tools`（工具门控）、`model`、`effort`、`context: fork`（subagent 隔离）、`agent: Explore|Plan|general-purpose|<custom>`、`hooks`、`paths`、`shell`。
4. **动态 context 注入**：`` `!cmd` `` 在加载时跑、stdout 替换占位符进入 prompt（fenced 形式 ``` ```! ```）；可在 settings 用 `disableSkillShellExecution: true` 关闭（不影响 bundled / managed）。`$ARGUMENTS` / `$ARGUMENTS[N]` / `$N` / 命名参数支持，加 `${CLAUDE_SESSION_ID}` / `${CLAUDE_EFFORT}` / `${CLAUDE_SKILL_DIR}` 环境变量。
5. **内置 bundled skills**：`/code-review` / `/batch` / `/debug` / `/loop` / `/claude-api` 默认可用；`/run` / `/verify` / `/run-skill-generator` 需 v2.1.145+，按项目类型（CLI、server、TUI、browser-driven）自动推断启动方式。

## Brief（梳理）

### 4 级加载范围（Scope Chain）

| 层级 | 路径 | 优先级 |
|---|---|---|
| **enterprise** | 组织级（managed）| 最高 |
| **personal** | `~/.claude/skills/` | 中 |
| **project** | `.claude/skills/` | 中低 |
| **plugin** | `plugin-name:skill-name` 命名空间 | 最低（plugin 隔离）|

- **冲突解析**：高优先级覆盖低优先级；同层级按字母序
- **plugin namespace**：`plugin-name:skill-name` 格式——避免跨 plugin 命名冲突

### SKILL.md 16 字段 Frontmatter 详表

| 字段 | 类型 | 用途 |
|---|---|---|
| `name` | string | kebab-case 标识（与目录名一致）|
| `description` | string | **触发器不是摘要**——Claude 据此判定何时加载 |
| `when_to_use` | string | 显式触发场景（比 description 更具体）|
| `argument-hint` | string | 提示用户参数格式，如 `[branch-name]` |
| `arguments` | array | 形式化参数定义 |
| `disable-model-invocation` | bool | **user-only**——Claude 不能自动调，只允许 user 手动 `/` |
| `user-invocable` | bool | **model-only**——Claude 自动调，user 看不到 |
| `allowed-tools` | string | 工具白名单（预批，不需 permission）|
| `disallowed-tools` | string | 工具黑名单（即便 model 想用也禁）|
| `model` | string | 强制走特定模型（sonnet / opus / haiku）|
| `effort` | enum | 思考深度（low / medium / high）|
| `context: fork` | bool | **subagent 隔离**——在独立 context 跑 |
| `agent` | enum | subagent 类型（Explore / Plan / general-purpose / 自定义）|
| `hooks` | object | 局部 hook 配置 |
| `paths` | array | 文件 glob 模式（限定 skill 适用目录）|
| `shell` | string | 强制 shell 解释器（bash / zsh）|

- **关键对立**：`disable-model-invocation: true` vs `user-invocable: false`
  - 部署命令 = user-only（避免 Claude 误触发）
  - 后台工具 = model-only（user 不知情但自动执行）

### 动态 Context 注入

| 机制 | 语法 | 行为 |
|---|---|---|
| **Command 注入** | `` !`git status` `` | 加载时跑，stdout 替换进 prompt |
| **fenced 形式** | ```` ```!``` ```` | 同上但更显眼 |
| **环境变量** | `${CLAUDE_SESSION_ID}` | session 唯一 ID |
| **环境变量** | `${CLAUDE_EFFORT}` | 当前 effort 等级 |
| **环境变量** | `${CLAUDE_SKILL_DIR}` | skill 所在目录绝对路径 |
| **关闭** | settings 加 `disableSkillShellExecution: true` | 不影响 bundled / managed |

- **风险**：`!cmd` 在 SKILL.md 加载时跑——可被恶意 SKILL.md 利用做 prompt injection
- **缓解**：始终设置 `disableSkillShellExecution: true`，只在 trusted skill 才用

### 参数与命名参数

| 形式 | 用途 |
|---|---|
| `$ARGUMENTS` | 整个参数字符串 |
| `$ARGUMENTS[N]` | 第 N 个参数（0-indexed）|
| `$N` | 简写 = `$ARGUMENTS[N]` |
| **命名参数** | `--branch=main` → `$branch` |

- 例：`/deploy --env=prod --branch=main` → SKILL.md 里 `$env` = `prod`, `$branch` = `main`
