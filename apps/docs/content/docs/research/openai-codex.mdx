# openai-codex

## 一句话定位

> OpenAI 本地 coding agent CLI/TUI（Rust workspace），把 **sandbox** 与 **approval policy** 解耦成双轴——CC 最直接同源竞品。

## 关键 takeaway

1. **同源竞品**：CC TS 单体；Codex Rust workspace（~100 crate、CLI main.rs 3960 行、AGENTS.md 强制 module &lt;500 LoC）。`(methodology:L1)`
2. **Sandbox × Approval 双轴**：sandbox 三档 `read-only`/`workspace-write`/`danger-full-access`；approval 五态含 `granular`（按 prompt 来源分项开关）——比 CC 单维 `--permission-mode` 细。`(methodology:L2)`
3. **多模型一等公民**：`ollama`/`lmstudio`/`models-manager` 独立 crate；本地模型 first-class。
4. **Hooks 对标 CC**：10 个 event 命名一致，每 event 独立 JSON schema。
5. **Skills/Sub-agents/Memory 全做**：内置 system skills（`skill-creator`/`skill-installer`/`plugin-creator`）、`agent-graph-store` 子 agent 图、`memories` 异步两阶段 pipeline。
6. **三层分发**：CLI/TUI、`app-server` protocol（v2 active）、Python/TS SDK（stdin/stdout JSONL spawn CLI）。

## 与 Claude Code 的对比

- **栈**：CC Node/TS 单体 vs Codex Rust workspace。
- **Approval**：CC `--permission-mode` 单轴 vs Codex 双轴 + `granular` 子项——企业 admin 更细。
- **多模型**：CC 默认只走 Anthropic；Codex 把 ollama/lmstudio 当 first-class。
- **登录**：CC Anthropic API key；Codex 默认 **Sign in with ChatGPT**（复用 Plus/Pro 额度）。
- **共享概念**：hooks/skills/sub-agents/MCP/`AGENTS.md`/sandbox 都在，但 Codex schema 是 OpenAI 自定义，与 CC 不互通。

## 与 Claude Code 教程的关联

- **视角映射**：CC plan mode ≈ Codex `untrusted` + `workspace-write`；CC `--dangerously-skip-permissions` ≈ Codex `danger-full-access` + `never`。
- **教学素材**：sandbox × approval 双轴在企业部署讲解里比 CC 单轴清晰，可作"为什么需要分两层"案例。
- **反向差异**：CC plugin/MCP marketplace 比 Codex `mcp-server`（仅 `McpConnectionManager`）成熟；讲"MCP 工具产品化"仍以 CC 为主线。
- **Sub-agent 选型**：CC `Task` tool 无状态 vs Codex `agent-graph-store` 持久化图。

## Brief（梳理）

### Codex 定位：OpenAI 出品的 CC 同源竞品

- **栈差异**：CC Node/TS 单体 vs Codex Rust workspace（~100 crate）
- **CLI 体量**：main.rs 3960 行；**AGENTS.md 强制 module &lt;500 LoC**（CC 没这个限制）
- **License**：Apache-2.0

### Sandbox × Approval 双轴（核心设计）

| 轴 | 档位 / 状态 |
|---|---|
| **Sandbox** | `read-only` / `workspace-write` / `danger-full-access`（三档）|
| **Approval** | `untrusted` / `on-failure` / `on-request` / `never` / **`granular`**（五态） |

- **`granular`** 子项：按 prompt 来源（user / skill / sub-agent）分项开关
- **对比 CC**：CC `--permission-mode` 单轴；Codex **双轴 + `granular` 子项**——企业 admin 更细
- **视角映射**（CC plan mode ↔ Codex）：
  - CC plan mode ≈ Codex `untrusted` + `workspace-write`
  - CC `--dangerously-skip-permissions` ≈ Codex `danger-full-access` + `never`

### 多模型一等公民

- **独立 crate**：`ollama` / `lmstudio` / `models-manager`
- **本地模型 first-class**——CC 默认只走 Anthropic API
- **场景**：企业内网 / 数据合规 / 成本敏感

### Hooks / Skills / Sub-agents / Memory 全做

| 概念 | Codex 实现 |
|---|---|
| **Hooks** | 10 个 event 命名**与 CC 一致**；每 event 独立 JSON schema |
| **Skills** | 内置 `skill-creator` / `skill-installer` / `plugin-creator` system skills |
| **Sub-agents** | `agent-graph-store` 持久化**图**（vs CC `Task` tool 无状态）|
| **Memory** | `memories` **异步两阶段 pipeline**（vs CC 同步 auto memory）|

- **共享概念，schema 不互通**——同一名字不同 JSON 结构（OpenAI 自定义）

### 三层分发形态

| 形态 | 形态描述 |
|---|---|
| **CLI / TUI** | 终端交互 |
| **`app-server` protocol** | v2 active（与 CC 类似，但不是 MCP）|
| **Python / TS SDK** | stdin/stdout JSONL spawn CLI（包装 CLI，**不是 native**）|

- 与 CC 类似：CLI + SDK 两条路；但 Codex SDK 是 spawn CLI 而非 native

### 登录与商业化

- **默认登录**：**Sign in with ChatGPT**（复用 Plus / Pro 额度）
- 对比 CC：CC 默认 Anthropic API key
- **影响**：Codex 适合 OpenAI 现有用户零成本上手；CC 适合 Anthropic API 现有用户
