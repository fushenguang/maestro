# cursor-changelog

## 一句话定位

> Cursor 是 IDE-native 商业 coding agent（Anysphere 出品）；2026-05 至 2026-06 一系列 changelog 显示其走向：Composer 2.5 提速 / Bugbot 自动评审 / Canvas 多人协作 / Design Mode 视觉标注 / Enterprise Org 多团队治理 / SDK 工具生态扩展。

## 关键 takeaway

1. **Composer 2.5 提速 + 降价**：训练进展让 Bugbot 提速 3x（90s vs 5min）、便宜 22%、多抓 10% bug（0.62 vs 0.56） (blog:L5) (blog:L7)。
2. **/review 推送前自检**：`/review` 可在 git push 前跑 Bugbot + Security Review，GitHub/GitLab 上同一 diff 不会重复审 (blog:L11) (blog:L13)；Cursor 3.7+ 可用，CLI 即将上线 (blog:L15)。
3. **Canvas Design Mode 视觉化编辑**：在 canvas 上点选/标注 UI 元素代替文字描述，agent 看到选中元素 + 代码 + 布局关系 (blog:L13)。
4. **Context Usage Report**：在 canvas 中以可交互报表拆解 token 来源（system prompt / tool defs / rules / skills），可点 "Debug with Agent" 让 agent 找削减点 (blog:L19) (blog:L21)。
5. **Design Mode 多选 + 语音**：浏览器端多选元素让 agent 一次性改一组；mic 在 agent 运行中保持可用，可排队下一条语音指令 (blog:L9) (blog:L15)。
6. **Enterprise Organizations GA**：组织→团队→组三层结构；用户可跨多团队，最宽松权限生效 (blog:L9) (blog:L19)；含多团队成员、Org 级 IDP / 用量分析 / 跨团队调人 / 新人继承设置 (blog:L25-L33)。
7. **SDK 自定义工具 + auto-review**：`local.customTools` 走内置 MCP server `custom-user-tools`，无需自建 stdio/HTTP MCP (blog:L9) (blog:L11)；`local.autoReview` 用自然语言 `permissions.json` 描述 allow/block (blog:L19-L28)。
8. **SDK 持久化与子代理**：JSONL store 可读/diff/进版本控制；自定义 `LocalAgentStore` 接口可接 Postgres 或内存；subagent 可任意嵌套 (blog:L32) (blog:L38)。
9. **Shared Canvases + /loop**：canvases 可生成只读链接供团队浏览（Pro/Teams/Enterprise） (blog:L11)；`/loop` 让 agent 按本地调度重复跑直到达成目标 (blog:L17)。

## 与 Claude Code 的对比

- **载体差异**：Cursor IDE-native（编辑器内 Canvas + 可视化）+ 浏览器；Claude Code terminal-native + 文件工具。Cursor 把产物做成「可分享的工件」，CC 仍以 transcript + 文件为主。
- **多文件编辑**：Composer 2.5 一次任务可跨文件编排并改 UI（点选/语音驱动） (blog:L7) (blog:L13)；CC 用 Edit/Write 工具串联 + sub-agent 分任务，靠 prompt 工程协调。
- **评审闭环**：Bugbot = `/review`（推送前 + PR 上识别同 diff 跳过） + Composer 2.5 提速 + Enterprise 可治理；CC 靠 review skill / sub-agent prompt / hook（PostToolUse 跑 lint/test），无产品化「评审 agent」概念。
- **多人协作**：Cursor Canvas 链接 + Dashboard 只读 (blog:L11)；CC 无原生多人 UI，协作靠 git / PR。
- **企业治理**：Cursor Enterprise 提供 Org/Team/Group 三层 + IDP + 用量钻取 (blog:L25-L33)；CC Enterprise 主要靠 SSO + usage policy + audit log，无 Canvas/Dashboard 共享层。
- **工具生态**：SDK `customTools` 走 MCP server `custom-user-tools` (blog:L9)，与 CC 的 MCP 客户端地位对等；但 CC 的 MCP 客户端是用户在 settings 里挂载，Cursor 把 MCP server 内置化以简化 onboarding。

## 与 Claude Code 教程的关联

- 教程第 3 章 sub-agent 与第 5 章 hooks，可对照 Cursor 的 `/review` 推送前 hook + Bugbot subagent 思路（CC 用 hook 实现同效）。
- 教程第 6 章 MCP，对照 `customTools` 自动走 MCP server 的设计——CC 用户需要显式 `claude mcp add`，Cursor 把它包成 SDK 默认路径。
- 教程第 7 章 企业部署，对照 Enterprise Organizations 的三层 RBAC，CC 企业版目前主要靠 SSO + 策略文件，没有等价 Dashboard。

## Brief（梳理）

### 7 个 changelog post 详解

#### Post 1: Bugbot Updates（Jun 10）

- **Composer 2.5 训练进展**驱动 Bugbot：
  - 速度：**~90s vs ~5min**（3x 提速）
  - 成本：**-22%**/run
  - 抓 bug 率：**0.62 vs 0.56**（+10%）
- **`/review` 推送前自检**：可选 Bugbot / Security Review 组合；GitHub / GitLab 上同 diff 不重复审
- **Cursor 3.7+ + cursor.com/agents**；CLI coming soon
- **增量 review 模式**：只审自上次 review 以来新增的 diff

#### Post 2: Canvas Improvements（Jun 4, 3.7）

- **Design Mode in canvases**（前仅 browser）：点选 / 标注 UI 元素代替文字描述
- **Context Usage Report**（canvas 内可交互报表）：拆解 token 到 system prompt / tool defs / rules / skills；可点 **Debug with Agent** 让 agent 找削减点
- **Shared canvases 全屏** + agent 嵌入可点击 button + canvas type error 修复 + chart 自定义

#### Post 3: Design Mode Improvements（Jun 5, 3.7）

- **多选元素**：浏览器中点选多个 → agent 一次性改一组
- **语音输入** + **agent 运行中 mic 保持可用**——可排队下一条指令

#### Post 4: Enterprise Organizations（Jun 3, GA）

| 层 | 角色 |
|---|---|
| **Organization** | 顶层容器——身份 / 管理 / 成员 + 跨团队 spend rollup |
| **Team** | 部门 / 区域 / 子公司的运营单位（迁移现有 cursor team 进来）|
| **Group** | 轻量用户集合，跨 team 存在；同 user 多 team 时**最宽松权限生效**|

**5 项 GA 改进**：
- 多 team 成员资格
- Org 级 IDP 管理
- Org 级 usage analytics（钻取到 team）
- Admin 通过 dashboard / API / CSV 跨 team 调人
- 新成员**自动继承** team 设置

#### Post 5: SDK Updates（Jun 4）

- **`local.customTools`**：把自定义 function 透传给 local agent；走**内置 MCP server `custom-user-tools`**——与官方 MCP tool **同权限门**；subagent 自动可见
- **`local.autoReview`**：headless 跑默认无 approval；开启后 classifier 用自然语言 `permissions.json` 判定 allow/block
  - `autoRun.allow_instructions` / `block_instructions` 自然语言描述
  - 例子：read-only build artifacts 走 allow，delete 永远 hold
- **JSONL store + custom store**：替换 SQLite；JSONL 可读 / diff / 进版本控制；`LocalAgentStore` 接口可接 Postgres / 内存
- **Nested subagents**：subagent 可再 spawn subagent，每层独立 prompt + model；无需开启
- **Reliability**：run correlation `requestId` / local `wait()` 可靠化 / HTTP/1.1 兼容 / 轻量 import / 自包含 .d.ts / bundled ripgrep
- **Models**：`composer-2` slug 自动路由到 Composer 2.5

#### Post 6: Shared Canvases + /loop（May 20, 3.5）

- **Shared canvases**：生成只读链接给团队浏览（Pro / Teams / Enterprise）
- **`/loop` skill**：本地调度重复跑——agent 决定何时唤醒或响应事件；用例"check deploy status every 5 minutes" / "work on this feature until tests pass"

### 6 维 CC vs Cursor 决策表

| 维度 | Claude Code | Cursor | 谁更适合 |
|---|---|---|---|
| **载体** | terminal-native + 文件工具 | IDE-native（Canvas + 可视化）+ 浏览器 | 团队非工程角色 → Cursor；纯后端 / 远程 ssh → CC |
| **多文件编辑** | Edit/Write 工具串联 + sub-agent 分任务 | Composer 2.5 一次任务跨文件 + 语音/点选驱动 | 视觉密集 UI 改 → Cursor；复杂逻辑改 → CC |
| **评审闭环** | review skill / sub-agent prompt / hook | Bugbot = `/review`（推送前 + PR 同步识别同 diff 跳过）| 企业级 review pipeline → Cursor；定制化策略 → CC |
| **多人协作** | 无原生 UI，协作靠 git / PR | Canvas 链接 + Dashboard 只读 | 非技术 stakeholder review → Cursor；纯工程协作 → CC |
| **企业治理** | SSO + usage policy + audit log | Org/Team/Group 3 层 + IDP + 用量钻取 | 多子公司 + 复杂 RBAC → Cursor；简单 SSO → CC |
| **工具生态** | MCP 客户端（用户 settings 挂载）| SDK `customTools` 自动走 `custom-user-tools` MCP | 自建工具门槛低 → Cursor；多 MCP 集成灵活度 → CC |

- **关键差异**：Cursor 把 MCP server **内置化**以简化 onboarding；CC 走「用户显式 `claude mcp add`」路径
- **本质区分**：**载体不同**——CC 是 transcript + 文件；Cursor 是「可分享的工件」
