# anthropics/anthropic-cookbook（官方 SDK cookbook）

> **Cookbook · 官方 code-recipe 集合**——copy-paste 的 notebook recipe，按能力分类，无教学叙事。

## 速览

- **Stars**：~45.3k · **License**：MIT
- **Notebook**：87 `.ipynb` + 83 `registry.yaml` 登记
- **栈**：Python · `uv` · ruff · Claude AI Review

## 目录

| 目录 | 用途 |
|---|---|
| `capabilities/` | RAG / 分类 / 摘要 / SQL |
| `tool_use/` | 工具调用（19 nb） |
| `claude_agent_sdk/` | Agent SDK 7 demo |
| `managed_agents/` | Managed Agents |
| `skills/` | Skill 范式 |
| `third_party/` | 第三方（Pinecone · Voyage · LlamaIndex · MongoDB） |

## 范式

1. **Notebook-first**：每 recipe = 1 `.ipynb`，输出 cells 保留作 expected results
2. **Registry 驱动**：`registry.yaml` 单一真相源
3. **One concept per notebook**（`CLAUDE.md` 明文）
4. **无教学叙事**：与 `anthropics/courses`（章节递进）/ Claude Code（CLI 指南）根本不同
5. **演示警示**：cell 标 `for demonstration purposes only`

## 何时用

- 找官方代码：RAG / 分类 / 工具调用 / 视觉 / 缓存
- 找第三方集成：Pinecone / Voyage / LlamaIndex
- 找 agent 脚手架：chief-of-staff / SRE / threat intel
- **`registry.yaml` 是「recipe 集合如何做内容索引」的样板**

## 与 Claude Code 教程的关联

- cookbook 教 **Claude API**（不是 Claude Code CLI）——两条独立轨道；教程可作为「先学 API 再学 CLI」的入门跳板
- 「one concept per notebook」是教程章节切分的反面参考——教程要讲叙事递进，而 cookbook 要 atomic 可复制；区分场景

## 不适用

- 学 Claude API 基础 → `anthropics/courses`
- 学 Claude Code → 那是另一个 repo
- production 部署 → notebook 自带 `for demonstration only` 警告
